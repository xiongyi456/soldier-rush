# 士兵冲锋 3D：豆丁的司令之路

原创战斗 Q 版单人 3D 冲锋游戏。项目使用 Vite、TypeScript 和 Three.js，可作为网页/PWA运行，也已准备好通过 Capacitor 封装 Android 与 iOS。

## 当前玩法

- 只控制一名主角「豆丁」（Q 版大头 chibi），自动射击，玩家通过键盘、鼠标或触屏左右移动。
- 从新兵、列兵、下士一路晋升到将军和司令，共 13 级军衔。
- 每次晋升暂停战斗，从攻击、防御、战术/支援三张技能卡中选择一项。
- 武器在关键军衔进化：步枪、冲锋枪、霰弹枪、狙击枪、火箭筒、激光枪。
- **当前武器决定射速**（不再继承历史最快射速）；多弹道逐步加扇形覆盖，后期不会一键满路清场。
- 只有火箭会爆炸溅射；激光/狙击保持穿透，主命中更痛、溅射做补刀。
- 主角拥有独立生命和护甲：护甲只吸收部分伤害，护甲耗尽后仍可战斗，生命归零才会结束游戏。Boss 技能会优先造成明显的生命伤害。
- 武器采用分阶段总火力预算；小怪血量随弹道数/火力阶段补偿，避免升级后秒杀全场。
- **远程锁定枪兵**：打完前 **2 个 Boss** 后才出现；锁定圈约 0.9 秒，及时换位可完全躲开。
- 武器箱打破后掉落**奖励核心**，靠近即大范围磁吸并自动拾取（多箱并排也能一次扫光）。
- 道路上会出现地刺、延时地雷和电磁减速区，陷阱保留可躲避通道，不会挡住子弹。
- **Boss 不再按固定 500m 刷新**，改为击杀数 **或** 军衔门槛，先到先召唤：

| 战役 Boss | 名称 | 击杀 | 或军衔 |
|-----------|------|------|--------|
| 1 | 铁罐头大佐 | 28 | 下士(3) |
| 2 | 铁饼队长 | 62 | 上士(5) |
| 3 | 红点幽灵 | 105 | 中尉(7) |
| 4 | 爆米花将军 | 155 | 少校(9) |
| 5 | 棱镜哨兵 | 210 | 上校(11) |

- 达标后约 0.9 秒预警再入场；HUD 显示「再杀 X / 升某某」。
- Boss 招式会预留可读安全带，避免整条路被封死。
- 击败第 5 个 Boss 后可选**战役胜利结算**或**无尽冲锋**（无尽按再击杀一批怪召唤下一战）。
- 天气在阴天、小雨、薄雾和黄昏之间变化，只影响画面。
- 司令功勋圆满后可以授勋转生。本局军衔、武器与技能会重置，永久司令勋章保留。

## 技术结构

- Vite + TypeScript 严格模式
- Three.js ES Modules
- GLTFLoader、AnimationMixer、Meshopt、KTX2 资源管线
- EffectComposer 与分级辉光效果
- vite-plugin-pwa 离线缓存
- Capacitor Android/iOS 壳与原生存档适配
- 对象池复用弹道与短命特效（击杀不再堆满屏四面体碎片）

当前仓库没有附带正式商业 GLB 角色模型，因此默认使用程序化 Q 版角色。将符合规范的 `hero.glb` 放到 `public/assets/models/` 后即可接入骨骼模型；详细节点和动画命名见该目录的说明文件。

## 本地运行

需要 Node.js 与 npm。

```powershell
npm install
npm run dev
```

浏览器打开终端显示的地址。也可以在 Windows 中直接运行：

```powershell
./start-mobile-server.ps1
```

## 手机同一 Wi-Fi 试玩

```powershell
./start-mobile-server.ps1 -Port 8080
```

手机和电脑连接同一个 Wi-Fi，然后打开脚本显示的局域网地址，例如 `http://192.168.1.5:8080`。

## 构建与预览

```powershell
npm run build
npm run preview
```

生产文件生成在 `dist/`。PWA manifest 和 Service Worker 会由构建自动生成，不再手工维护缓存版本。

## Android / iOS

仓库已经包含 `android/` 和 `ios/` 平台工程。修改 Web 游戏后同步构建：

```powershell
npm run cap:sync
```

打开原生工程：

```powershell
npm run cap:android
npm run cap:ios
```

iOS 工程需要在 macOS/Xcode 中构建。Android 需要 Android Studio 和可用的 SDK。

## 操作方式

- 电脑：方向键、A/D 或鼠标拖动。
- 手机：按住画面左右滑动。
- 技能选择：点击技能卡，电脑也可按数字键 1、2、3。
- 状态按钮：查看军衔、武器、技能、生命、护甲、勋章、Boss 进度和转生状态。

## 存档兼容

- Web 使用 localStorage。
- Capacitor 原生版本使用 Preferences，并与 Web 存档适配器保持相同结构。
- v1 存档会自动迁移到 v2，保留已解锁武器、最高 Boss、最高分与最远距离。
- v2 新增司令勋章、转生次数和画质设置。

## 性能策略

- 中端手机以 60 FPS 为目标，低性能设备降低像素比、粒子、雨滴和后处理。
- 粒子上限收紧，击杀反馈以短火花/光环为主，避免长期残留碎片。
- 主角正式模型建议 15k–25k 三角面、1K 纹理、70 根以内骨骼。
- 核心首屏资源建议压缩后控制在 12 MB 以内，Boss 模型按需加载。
